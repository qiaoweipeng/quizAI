# 消防-理论考试系统

​	消防理论考试系统是一款专为消防设施操作员资格考试设计的在线学习与模拟考试平台。系统提供多种考试模式，帮助考生高效备考，提升考试通过率。

## 系统定位

- **目标用户**: 消防设施操作员（监控方向）备考人员
- **核心价值**: 提供真实的模拟考试环境，帮助考生熟悉考试形式和题型
- **数据来源**: 基于鉴定站最新真题和考试大纲构建题库

## 系统功能

### 1. 考试功能

系统提供两大核心模式，满足不同备考需求：

**考试模式**

| 模式 | 说明 | 题目构成 | 评分标准 |
|------|------|----------|----------|
| **固定考卷** | 题目固定不变，适合模拟真实考试环境 | 100道单选 + 40道多选 + 60道判断 | 每题0.5分，满分100分，60分合格 |
| **随机考卷** | 题目随机抽取，每次考试内容不同 | 同上 | 同上 |

**练习模式**

| 模式 | 说明 | 题目构成 | 评分标准 |
|------|------|----------|----------|
| **专项练习** | 针对特定题型进行强化训练 | 自定义题型（单选/多选/判断）和数量（10题~全部） | 无分数，显示正确率（如答对9/10题，正确率90%） |

> **考试时间**：固定卷和随机卷限时90分钟，专项练习不限时  
> **题库更新**：试题每日根据鉴定站最新真题实时更新

### 2. 界面设计

- **设计风格**：简约清爽，专注答题体验
- **设备适配**：目前仅适配PC端，建议使用桌面浏览器访问
- **答题界面**：提供两种视图模式，可自由切换
  - **单题模式**：每次展示一道题，支持手动切换或答后自动跳转
  - **全卷模式**：展示全部题目，通过滚动条浏览

### 3. 辅助功能

- **即时反馈**：三种模式均可选择是否答完立即显示对错及解析
- **错题回顾**：考试结束后自动汇总错题，附带详细解析
- **一键预答**：考试模式支持快速预答（单选默认选A，多选全选，判断选对），方便测试流程

## 题目数据格式

题目数据存储在 `public/json/exam-papers.json` 文件中，格式如下：

```json
{
  "papers": [
    {
      "id": "1",
      "name": "2026年中级消防设施操作员[监控]历年真题练习（卷A）",
      "questions": [
        {
          "id": "dx-1",
          "type": "single",
          "question": "题目内容",
          "options": [
            "A. 选项1",
            "B. 选项2",
            "C. 选项3",
            "D. 选项4"
          ],
          "answer": ["A"],
          "parse": "解析内容"
        }
      ]
    }
  ],
  "questions": []
}
```

### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `type` | string | 题目类型：`single`(单选)、`multiple`(多选)、`judge`(判断) |
| `question` | string | 题目内容 |
| `options` | array | 选项数组 |
| `answer` | array | 正确答案数组，如 `["A"]` 或 `["A","B"]` |
| `parse` | string | 解析内容 |

## 添加新题目

1. 在 `public/json/` 文件夹中创建或编辑 JSON 文件
2. 按照上述格式添加题目数据
3. 重启开发服务器或刷新页面即可加载新题目

## 注意事项

- 题目文件必须放在 `public/json/` 文件夹中
- JSON 文件名建议使用英文，避免编码问题
- 确保题目 ID 唯一，避免重复
- 目前系统仅适配 PC 端，建议使用桌面浏览器访问

## 开发说明

- 使用 pnpm 作为包管理器，不支持 npm
- 题目数据通过 fetch API 从 public 文件夹加载
- 支持热更新，修改代码后自动刷新
- 项目已配置 ESLint 和 Prettier 代码规范

## 部署说明

### GitHub + Netlify 部署

1. 将项目推送到 GitHub 仓库
2. 登录 Netlify，点击「New site from Git」
3. 选择你的 GitHub 仓库
4. 配置构建命令：`pnpm build`
5. 配置发布目录：`dist`
6. 点击「Deploy site」完成部署

### 其他部署方式

```bash
# 构建生产版本
pnpm build

# 使用静态服务器部署
cd dist
npx serve
```
